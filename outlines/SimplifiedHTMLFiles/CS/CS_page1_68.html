Course Description

CS 436:

An introduction to networks, protocols, and distributed systems. Layered models, resource management, naming, addressing and routing, reliable communication, security, and higher-level services.

View requirements for CS 436

CS 636:

An introduction to computer networks, Internet protocols, and distributed systems. Layered models, naming, addressing and routing, reliable communication, security, and elements of distributed system design. Master of Health Informatics students only.

View requirements for CS 636

CS 436 provides an introduction to computer networks, protocols, and distributed system architectures for students who are not computer science majors. It provides an introduction to principles, algorithms, protocols, and technology standards used in computer networks and distributed systems. It is intended for 3rd- or 4th-year students with an interest in Computer Science and is not open to Computer Science students. It will be held with CS 636.

Prospective students should be able to:

  • Write simple imperative programs using lists, iteration, and array indexing for both read and write (mutable memory model).
  • Describe data representations used by computer hardware at the bit level, operate on these representations, and compute their values.
  • Explain the basic build process for computer software.
  • Exemplify the basic functionality and components of an operating system.

 

Learning Outcomes

No explicit learning outcomes defined for this course.
  • Recall, explain, and compare transmission media and access control schemes.
  • Recall, explain, and compare addressing and message forwarding paradigms, including path computation.
  • Recall, summarize, and exemplify connection management and reliable message transmission.
  • Recall, explain, and compare naming and name resolution techniques, including mobility.
  • Recall, summarize, and exemplify contemporary network protocols and techniques.
  • Recall, summarize, and explain security/privacy concepts in network protocols and distributed systems
  • Recall, explain, and compare high-level communication services and distribution services.
  • Compare how existing distributed applications use network services.

 

Tentative Course Schedule

We will take a ‘top-down’ approach to networking and distributed systems, focusing on the networking concepts and distributed systems mechanisms underlying widely used networked applications and systems.

  1. Introduction: Internet topology, network edge, network core, performance, protocol layers. (remove OR move security from slides)
  2. Application layer
    1. Architecture: client-server, peer-to-peer (P2P)
    2. Networked applications: Web, remote login, email, file sharing, VoIP, video streaming, etc.
    3. Application layer protocols: HTTP, SSH, SMTP, DNS, BitTorrent, CDN, etc.
    4. Socket programming.
  3. Transport layer
    1. Overview: TCP vs. UDP semantics, segment format, multiplexing, connection management.
    2. Reliable data transfer, flow control, congestion control, rate control.
  4. Network layer
    1. Datagrams, network layer services, IP header, IP addresses, fragmentation and reassembly.
    2. Routing principles: Link state and distance vector algorithms, hierarchical routing, routing among autonomous systems.
    3. Routing protocols: OSPF, RIP, BGP.
    4. Address allocation and local routing: NAT, DHCP, etc.
  5. Link Layer: LANs, framing, error correction, MAC address, ethernet, ARP, CSMA/CD.
  6. Security
    1. Principles: confidentiality, integrity, cryptography, keys.
    2. Protocols.
  7. Distributed systems
    1. Design principles: End-to-end principle, hierarchy, replication and failover, etc.
    2. Implementation algo: processes, threads, RPC, transactions, publish/subscribe.

Assignments
There will be four homework assignments, and four quizzes. Each homework assignment or quiz will test the material taught in the most recent lectures. Homework assignments and quizzes will be posted on LEARN, must be submitted on LEARN, and will be marked on LEARN. Late submissions / retakes / exemptions will not be accepted.

There will also be two more programming assignments. While the assignments are fairly large, you will have one month to complete each. Try not to leave these until the last minute. Programming assignments are to be returned by the provided due dates. In the case of illness or extraordinary circumstances, a verification of illness (VIF) must be submitted, and a 10% penalty will apply for each 24 hours late submission up to a maximum of 72 hours of the assignment deadline. No further extension will be granted. Programming assignments will be posted on LEARN, must be submitted on LEARN, and will be marked on LEARN.

Programming assignments should be completed individually. We will compare all handed-in assignments to each other to check for undocumented collaboration.

Administrative Policy

Plagiarism
A programming assignment is an individual creative process. Individuals must reach their own understanding of the problem and discover a path to its solution. During this time, discussions with friends are encouraged. However, when the time comes to write the code that solves the problem, such discussions are no longer appropriate; the program must be your own work.

Do not, under any circumstances, copy another person's program. This includes relevant web sources. Writing code for use by another or using another's code in any form is academic fraud and will be dealt with harshly. You are also responsible for ensuring that the code you write for the assignments is not readable by others.

Marking Appeals
Assignment appeals should be directed to the TA who marked the assignment. Exam appeals need to be submitted in writing to the instructor. The whole exam will be remarked.

The intellectual property rights owner deserves to know (and may have already given their consent). Read Policy 73 - Intellectual Property Rights.
